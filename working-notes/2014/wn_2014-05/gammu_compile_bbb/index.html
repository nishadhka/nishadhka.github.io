gammu compile BBB

2014-05-05

Gammu_compile-in-BBB

  • Gammu was compiled in BBB follwoing this
  • The dependacnies was intaalled using this command apt-get install cmake python-dev pkg-config libmysqlclient-dev libpq-dev \ libcurl4-gnutls-dev libusb-1.0-0-dev libdbi0-dev libbluetooth-dev \ libgudev-1.0-dev libglib2.0-dev unixodbc-dev
  • the command ./configure gives error of CMake Error: CMake was unable to find a build program corresponding to "Unix Makefiles".
  • based on this installed make by apt-get install make after this the error goes away
  • executed this command cmake .. -DCMAKE_INSTALL_PREFIX="/usr"
  • then make, says all build
  • then make test, says failure for smsd-dbi-sqlite3 and pther sqlite3 related thinks, but other thinks are passed, test results goes like this 99% tests passed, 5 tests failed out of 1116 Total Test time (real) = 609.54 sec The following tests FAILED: 444 - smsd-dbi-sqlite3 (Failed) 445 - smsd-inject-dbi-sqlite3 (Failed) 446 - smsd-inject-long-dbi-sqlite3 (Failed) 447 - smsd-daemon-dbi-sqlite3 (Failed) 1114 - py-smsd-testing (Failed) Errors while running CTest make: *** [test] Error 8
  • then sudo make install gives no error! seems
  • now comes good news root@arm:/home/debian/gammu-1.33.0/build# gammu [Gammu version 1.33.0 built 15:03:15 on May 5 2014 using GCC 4.6] Usage: gammu [parameters] <command> [options]
  • the usual testing comment gammu --identify throughs a error with unable to connect with phone Phone does not support enabled echo, it can not work with Gammu! As per this changed device address from /dev/ttyUSB0 to /dev/ttyUSB2, now it is working and the gammu-smsd also working making txt sms file inside inbox folder.
