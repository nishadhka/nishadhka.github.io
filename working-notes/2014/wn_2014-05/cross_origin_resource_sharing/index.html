Cross Origin Resource Sharing

2014-04-30

#Cross-OriginResourceSharing#

  • WHile working with qs into cbe-air, in populating table and charts with istsos json data, stucked with a error
  • the error was not informative in firebug.firefox, a simple correct repsonse of 200 with red fonts and also not giving any console.log in JS
  • after trying with changing jquery version, different old edited version of qs, experimenting with differneet json url no clue was found, finally tryed to run the aptana server in chromium and its devloper tools option gives verbal error of No 'Access-Control-Allow-Origin' header is present on the requested resource. Origin 'http://127.0.0.1:8020' is therefore not allowed access
  • this error is caused due to Same Origin Policy of browser to check foreign API in it to process.
  • to avoid this error is to use jsonp, proxy or CORS(Cross-Origin Resource Sharing), promising is CORS
  • In server CORS has to be enabled based on this and this
  • The comments are as follows sudo a2enmod headers to allow heaadrs in apache2
  • add this line to apache2.conf, inside Header set Access-Control-Allow-Origin "*"
  • check every thing is fine by apachectl -t and restart apache2 by sudo service apache2 reload
  • Follwoing this step in aws didnt removed the error, so follwoed this
  • Edited dafult file in /etc/apache2/sites-available/ and the line above istsos config line, and restarted the apache
  • Now the error is gone!
