Ubunut1204 recovery

2014-11-22

###Ubunut 12.04 recovery###

  1. It is found that changing the file /etc/environment has huge problem after reboot.
  2. Added the environment variables for ncl language in /etc/environment as export NCARG_ROOT=/usr/local/ncl-6.2.1; export PATH=$NCARG_ROOT/bin:$PATH.
  3. This was resultant in problematic reboot of the system. The system gets instead of into login page, gets flickering command log. By reboot went inside of command line and there no system command like ls or sudo was not working. saying no /bin folder in PATH environment. Indicating the problem of above.
  4. More over the command line is populated with info of KEYPAD ID some thing... with no room for enter ing anything. Still in this situation the command such as sudo nano /etc/environment was entered and removed those lines added for ncl language and then saved quickly. For running the sudo and nano its respective folder has to be entered to over come the no /bin PATH error.
  5. After this the system loaded without any error mentioned earlier.
